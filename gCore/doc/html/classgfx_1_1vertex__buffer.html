void gfx::vertex_buffer::upload_data ( )
virtual

Upload the buffer's data to OpenGL.

Todo:
There is no check to see if you have actually specified the data format. This means, internally, the stride member has not been correctly specified and so the upload will send OpenGL an unpredictable number of bytes (usually 0 but in more complex use cases it could be anything depending on the lifetime of the particular buffer). It also doesn't check to see if you have even loaded any data into the buffer client- side or if the buffer is "dirty". Though that is not quite as bad, it certainly isn't very helpful for the user.
